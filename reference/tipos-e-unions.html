Tipos básicos & unions

Anotar um tipo

let nome: string;        // qualquer texto
let total: number;      // inteiro ou decimal (um só number)
let aberta: boolean;    // true | false

Literal type — um valor exato vira um tipo

let camada: "migration";   // só aceita a string "migration"

Union | — "uma de"

type Profile = "modular" | "flat";   // perfil do repo (PRD §4.0)

| = "ou" (union, uma de várias). Não confundir com & = "e ao mesmo tempo".

Type alias — dar nome a um tipo

type Layer = "migration" | "model" | "outros";

Reutiliza o tipo por toda parte sem repetir a lista. Um type some em runtime.

Assinatura de função tipada

function categorize(path: string): Layer { /* … */ }
//                  └ parâmetro      └ tipo de retorno

Corpo de função: regras "primeira que casa vence"

if (path.endsWith("Test.php")) return "tests";   // tests ANTES de action
if (path.includes("/Actions/")) return "action";
return "outros";                              // balde: fecha o furo

.endsWith() / .includes() são métodos de string do JS, devolvem boolean.

Tipo vs. teste (a fronteira)

  • Tipo garante o formato: a saída é algum Layer. Sem balde final → erro "caminho sem retorno" (ts 2366).
  • Teste garante o comportamento: é o Layer certo. A ordem das regras é lógica — só teste pega.

Object type / interface — dar nome a uma forma

interface ChangedFile {
  path: string;
  layer: Layer;
  additions?: number;   // '?' = opcional
}

Descreve só o formato: sem classe, sem new, some em runtime. Cobra campos faltando, nome de campo errado e valor fora do tipo.

Array T[] e .map()

const files: ChangedFile[] = paths.map(function (path) {
  return { path: path, layer: categorize(path) };
});                                  // string[] → ChangedFile[]

T[] = "array de T". .map(fn) = novo array com fn em cada item (igual array_map / Collections ->map). Forma curta: paths.map(p => ({ path: p, layer: categorize(p) })) — parênteses no objeto são obrigatórios.

Switch exaustivo: never / assertNever

Obriga (em compilação) a tratar todos os membros de uma union. Adicionou uma 17ª camada? Todo switch que esqueceu dela para de compilar.

function assertNever(x: never): never {
  throw new Error("Camada não tratada: " + x);
}

function labelFor(layer: Layer): string {
  switch (layer) {
    case "migration": return "Migration";
    /* ... TODAS as 16 ... */
    case "outros":    return "Outros";
    default:           return assertNever(layer); // ✗ compila se faltar caso
  }
}

Por quê: no default, o TS estreitou layer ao que sobrou. Tratou tudo → sobra never (nenhum valor), e só never entra em assertNever. Faltou um caso → aquele literal não é never → erro ali. Guarda duplo: compilação (faltou caso) + runtime (exceção clara).

Analogia: é o botão "finalizar review" do próprio Skanner — só destrava quando todo arquivo da PR está marcado. PR ganha um arquivo novo → trava de novo.

As 16 camadas canônicas do Skanner (ordem de exibição)

migrationmodelenumdtopolicyactionservicejobeventlistenerobservernotificationlivewirebladetestsoutros

A ordem é regra de exibição (PRD §4.2), não da union. Uma union não tem ordem.
