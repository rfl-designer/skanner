Dar forma aos dados: interface e a lista de arquivos da PR

Onde o Layer para de ser solto e gruda num objeto que o app inteiro vai carregar.

Win de hoje

Você modela um arquivo da PR como um objeto tipado — interface ChangedFile { path; layer } — e transforma uma lista crua de caminhos na lista classificada que a tela de review vai exibir. Composição das 3 lições: Layer (0001) + categorize (0002) + a forma de hoje.

Até agora categorize devolve um Layer solto. Mas a PR não tem "um caminho" — tem dezenas, e de cada um o app precisa guardar duas coisas juntas: o caminho e a camada dele. Isso pede um objeto com forma fixa.

1. interface: dar nome à forma de um objeto

interface ChangedFile {
  path: string;
  layer: Layer;
}

Isso diz: "um ChangedFile é qualquer objeto que tenha um path string e um layer do tipo Layer — nem mais, nem menos". A partir daqui o compilador cobra essa forma em todo lugar que pede um ChangedFile.

Ponte com PHP Pense num DTO do Laravel — class ChangedFileData { public string $path; public Layer $layer; } — mas sem a classe. Uma interface de TS descreve só a forma: não tem construtor, não tem new, e some em runtime (igual aos outros tipos). É um contrato de formato, não um objeto que você instancia.

2. O compilador passa a cobrar a forma

Veja as três coisas que ele agora pega sozinho — todas em tempo de escrita:

const a: ChangedFile = { path: "x.php" };
// ✗ falta 'layer'. Property 'layer' is missing.

const b: ChangedFile = { paht: "x.php", layer: "model" };
// ✗ typo: 'paht' não existe em ChangedFile.

const c: ChangedFile = { path: "x.php", layer: "modle" };
// ✗ "modle" não é um Layer (herdou a checagem da 0001).

Forma faltando, nome de campo errado, valor fora da union — os três bugs mais comuns ao montar objetos, mortos antes de rodar. Em JS puro, os três passariam e quebrariam lá na frente.

3. Uma lista deles: ChangedFile[]

Os colchetes depois de um tipo significam "um array de". ChangedFile[] é "lista de arquivos com aquela forma" — exatamente o que a tela de review percorre. 

const paths: string[] = [
  "database/migrations/2024_create_plans_table.php",
  "app/Contexts/Crm/Actions/CreateUser.php",
  "tests/Feature/Actions/CreateUserActionTest.php",
];

4. O win: de caminhos crus → lista classificada

Montar { path, layer } na mão seria bobo — você já tem categorize. Use .map() para transformar cada caminho no objeto: 

const files: ChangedFile[] = paths.map(function (path) {
  return { path: path, layer: categorize(path) };
});
// files agora é a lista tipada que a UI vai agrupar e exibir.
Buraco de JS preenchido — .map() .map(fn) percorre um array e devolve um novo array com o resultado de fn em cada item (o original não muda). É o array_map() do PHP, ou o ->map() das Collections do Laravel. Aqui: cada string vira um ChangedFile, então string[] → ChangedFile[]. O compilador infere esse tipo de saída sozinho — e ainda confere que ele bate com o que você anotou.
A forma curta (vai aparecer muito no React) O mesmo .map com arrow function: paths.map(path => ({ path, layer: categorize(path) })). Dois detalhes: { path } é atalho para { path: path } (mesmo nome); e os (parênteses) em volta do objeto são obrigatórios — sem eles o { } vira bloco de código, não objeto. Por hoje, fique com a forma longa acima; só saiba reconhecer a curta.

Pratique (recuperação ativa)

Responda de memória antes de revelar.

P1 Com interface ChangedFile { path: string; layer: Layer }, o objeto { path: "x.php" } (sem layer) compila como ChangedFile?

A interface lista path e layer como obrigatórias. Faltou layer → erro "Property 'layer' is missing". Para tornar um campo opcional, usa-se layer?: Layer — mas aí é decisão sua, não o padrão.

P2 O que string[] significa?

T[] = "array de T". string[] é uma lista de strings; ChangedFile[] é uma lista de objetos com aquela forma. Os colchetes vêm depois do tipo do elemento.

P3 Uma interface em TypeScript serve para…

Interface descreve formato e nada mais: sem classe, sem new, e some em runtime (não valida nada quando o app roda — isso seria papel de um teste ou de uma checagem manual). É contrato em tempo de compilação.

Faça agora (5 minutos)

  1. Na Playground, junte tudo: type Layer + categorize (das 0001–0002) + a interface ChangedFile + o paths + o .map de hoje.
  2. Dê console.log(files) e confira na aba Logs que cada item tem path e o layer certo.
  3. Provoque os 3 erros do passo 2: remova layer de um objeto, troque path por paht, ponha um layer: "modle". Leia cada mensagem — é o compilador trabalhando por você.
  4. Passe o mouse sobre files e veja o TS dizer ChangedFile[] — tipo que ele inferiu, você não precisou repetir.
